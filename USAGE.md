# HLSenpai Usage Guide

This guide covers:

- Installing from GitHub Releases
- Building from source
- Installing required system dependencies (`ffmpeg`, codecs, and media libs)
- Running the app
- Encoding workflow
- AWS S3 upload workflow and credentials
- Config files and their schema
- Troubleshooting

## 1. Quick Start

If you want the fastest path:

1. Download a binary from the repository **Releases** page.
2. Install `ffmpeg` on your machine (with common codecs enabled).
3. Run `hlsenpai` (or `hlsenpai.exe` on Windows).
4. Select a video, open **Encode Options**, click **Encode**.
5. Optionally click **Upload to...** and push output to S3.

If no release fits your platform, build from source (see section 3).

## 2. Install From Releases

Release assets are generated by CI in this format:

- Linux/macOS: `hlsenpai-<version>-<platform>.tar.gz`
- Windows: `hlsenpai-<version>-<platform>.zip`

`<version>` is currently a UTC timestamp (for example `20260301-173055`).

### Linux/macOS

```bash
tar -xzf hlsenpai-<version>-<platform>.tar.gz
chmod +x hlsenpai
./hlsenpai
```

### Windows (PowerShell)

```powershell
Expand-Archive .\hlsenpai-<version>-<platform>.zip -DestinationPath .
.\hlsenpai.exe
```

Note: release archives currently package only the app binary. You still need runtime dependencies installed (especially `ffmpeg`; see section 4).

## 3. Build From Source

### 3.1 Prerequisites

- Rust stable toolchain (`rustc`, `cargo`)
- `ffmpeg` and development libraries
- GStreamer libraries (used by video preview stack)
- Platform build essentials (compiler, `pkg-config`, etc.)

### 3.2 Linux (Ubuntu/Debian)

The following package set mirrors CI:

```bash
sudo apt-get update
sudo apt-get install -y \
  build-essential \
  pkg-config \
  ffmpeg \
  libavcodec-dev \
  libavformat-dev \
  libavutil-dev \
  libavfilter-dev \
  libavdevice-dev \
  libswscale-dev \
  libswresample-dev \
  libgstreamer1.0-dev \
  libgstreamer-plugins-base1.0-dev \
  gstreamer1.0-plugins-base \
  gstreamer1.0-plugins-good \
  gstreamer1.0-plugins-bad \
  gstreamer1.0-libav \
  libasound2-dev \
  libx11-dev \
  libxrandr-dev \
  libxinerama-dev \
  libxcursor-dev \
  libxi-dev \
  libxkbcommon-dev \
  libwayland-dev \
  libudev-dev \
  libgtk-3-dev
```

### 3.3 macOS (Homebrew)

```bash
brew update
brew install ffmpeg gstreamer gst-plugins-base gst-plugins-good gst-plugins-bad gst-libav pkg-config
```

If `pkg-config` cannot find libraries:

```bash
export PKG_CONFIG_PATH="$(brew --prefix gstreamer)/lib/pkgconfig:$(brew --prefix gst-plugins-base)/lib/pkgconfig:$(brew --prefix ffmpeg)/lib/pkgconfig:$PKG_CONFIG_PATH"
```

### 3.4 Windows (MSVC)

CI uses `vcpkg` for FFmpeg and Chocolatey for GStreamer. A close local setup:

1. Install Rust MSVC toolchain and Visual Studio C++ Build Tools.
2. Install `vcpkg` and FFmpeg:
```powershell
git clone https://github.com/microsoft/vcpkg $env:TEMP\vcpkg
& "$env:TEMP\vcpkg\bootstrap-vcpkg.bat" -disableMetrics
& "$env:TEMP\vcpkg\vcpkg.exe" install ffmpeg:x64-windows
```
3. Set environment:
```powershell
$env:VCPKG_ROOT="$env:TEMP\vcpkg"
$env:VCPKGRS_TRIPLET="x64-windows"
$env:VCPKGRS_DYNAMIC="1"
```
4. Install GStreamer runtime/dev packages (for preview pipeline), then set `PKG_CONFIG_PATH` and `PATH` to your GStreamer install if needed.

### 3.5 Build and Run

Development build:

```bash
cargo run
```

Optimized build:

```bash
cargo build --release
./target/release/hlsenpai
```

Windows optimized binary:

```powershell
cargo build --release
.\target\release\hlsenpai.exe
```

## 4. Dependency Verification

### 4.1 Verify Rust

```bash
rustc --version
cargo --version
```

### 4.2 Verify FFmpeg

```bash
ffmpeg -version
```

Check expected encoders:

```bash
ffmpeg -hide_banner -encoders | rg "libx264|libx265|libvpx-vp9|libaom-av1|aac|libopus|libmp3lame"
```

If `rg` is unavailable, use:

```bash
ffmpeg -hide_banner -encoders | grep -E "libx264|libx265|libvpx-vp9|libaom-av1|aac|libopus|libmp3lame"
```

## 5. Using The App

### 5.1 Open Source Video

1. Launch app.
2. Click **Select File**.
3. Choose a video file (supported picker filter includes: `mp4`, `mov`, `mkv`, `m4v`, `avi`, `webm`, `ts`, `m2ts`).
4. The app validates the stream and shows metadata + preview controls.

### 5.2 Encode

1. Click **Encode Options**.
2. Adjust:
- Scale
- Codec/profile/preset/GOP/sc_threshold
- Audio codec/channels
- HLS packaging values
- Output folder + filenames
- Variant bitrate ladder
3. Click **Encode**.
4. Monitor progress in **Encode Output** modal.
5. Optional: click **Print ffmpeg Script** to inspect a shell script equivalent of current settings.

### 5.3 Upload To S3

After a successful encode:

1. Click **Upload to...**
2. Choose target (`AWS S3`)
3. Fill bucket/prefix
4. Set overwrite mode:
- `Overwrite existing`
- `Skip existing`
5. Click **Upload**

Upload logs show file counts, skipped/failed counts, and byte progress.

## 6. Encoding Defaults and Output Layout

Important defaults from the app:

- GOP: `2 * source_fps` (or `60` fallback)
- HLS segment duration: `6` seconds
- Playlist type: `vod`
- `independent_segments`: enabled
- Segment filename pattern: `v%v/seg_%06d.ts`
- Variant playlist pattern: `v%v/prog.m3u8`
- Master playlist file: `master.m3u8`
- Output base folder: `out`
- Default variants:
  - low: video `1200k`, maxrate `1500k`, bufsize `2400k`, audio `96k`
  - med: video `2500k`, maxrate `3000k`, bufsize `5000k`, audio `128k`
  - high: video `4500k`, maxrate `5400k`, bufsize `9000k`, audio `128k`

Resulting layout typically looks like:

```text
<output_base>/<output_subfolder>/
  master.m3u8
  v0/prog.m3u8
  v0/seg_000000.ts
  v0/seg_000001.ts
  v1/prog.m3u8
  ...
```

## 7. AWS Credentials and Access

HLSenpai supports two credential modes for S3 upload:

1. Explicit keys saved in app config
2. AWS default credential chain (leave access key and secret key empty)

In the credentials modal, **Access Key ID** and **Secret Access Key** must be both filled or both empty.

### 7.1 How To Get AWS Credentials

Recommended approach:

1. In AWS Console, go to IAM.
2. Create or choose an IAM user/role for uploads.
3. Attach least-privilege S3 permissions.
4. Generate Access Key for that user (if using static keys).
5. Put values into **Set Credentials** in the app.

Example minimal IAM policy for uploads to one bucket/prefix:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowListBucketPrefix",
      "Effect": "Allow",
      "Action": ["s3:ListBucket"],
      "Resource": "arn:aws:s3:::YOUR_BUCKET",
      "Condition": {
        "StringLike": {
          "s3:prefix": ["YOUR_PREFIX/*", "YOUR_PREFIX"]
        }
      }
    },
    {
      "Sid": "AllowObjectUploadAndRead",
      "Effect": "Allow",
      "Action": [
        "s3:PutObject",
        "s3:GetObject"
      ],
      "Resource": "arn:aws:s3:::YOUR_BUCKET/YOUR_PREFIX/*"
    }
  ]
}
```

Adjust policy to match your security model.

### 7.2 Using AWS Default Credential Chain

Leave Access Key + Secret Key empty in app credentials and click **Save**.

Then AWS SDK can resolve credentials from standard sources, including:

- Environment variables (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_SESSION_TOKEN`)
- Shared config/credentials files (`~/.aws/credentials`, `~/.aws/config`)
- IAM role credentials (EC2/ECS/EKS, etc.)

Region handling:

- If region is set in app credentials, that is preferred.
- Otherwise SDK default resolution is used.
- If still unresolved, app falls back to `eu-north-1`.

### 7.3 Important App Behavior

- `AWS S3` upload target appears only when an `aws` provider exists in auth config.
- Clicking **Set Credentials** and **Save** once creates that provider entry.
- You can save with empty key/secret if you want default credential chain.

## 8. Config Files

The app stores config in an OS-specific directory:

- If `XDG_CONFIG_HOME` is set: `$XDG_CONFIG_HOME/hlsenpai`
- Windows: `%APPDATA%\hlsenpai`
- Otherwise: `$HOME/.config/hlsenpai`
- Fallback (if path resolution fails): current working directory

Files:

- `auth_config.json` (credentials/provider settings)
- `encoding_presets.json` (last encode options)
- `upload_prefs.json` (last upload target/bucket/prefix/mode)

On Unix, `auth_config.json` is written with strict permission mode `600`.

If a config file becomes invalid JSON, the app renames it to `*.broken-<timestamp>.json` and regenerates defaults.

### 8.1 `auth_config.json` Example

```json
{
  "version": 1,
  "providers": {
    "aws": {
      "api_key": null,
      "access_token": null,
      "refresh_token": null,
      "extra": {
        "region": "us-east-1",
        "access_key_id": "AKIA...",
        "secret_access_key": "....",
        "session_token": "",
        "bucket": "my-output-bucket",
        "default_prefix": "vod/my-project"
      }
    }
  },
  "updated_at_utc": "1700000000"
}
```

Notes:

- `bucket` and `default_prefix` in `extra` are optional helpers for pre-filling upload form.
- Remove `access_key_id` + `secret_access_key` to rely on default credential chain.

### 8.2 `encoding_presets.json` Key Fields

Stored values include:

- Scale and aspect lock
- GOP and codec/preset/profile
- HLS packaging options
- Output folder and filename patterns
- Variant ladder settings

This file is automatically updated as users change encode options.

### 8.3 `upload_prefs.json` Key Fields

Stored values include:

- Last upload provider (for example `aws_s3`)
- Last overwrite mode (`overwrite` or `skip_existing`)
- Last bucket and prefix

## 9. Troubleshooting

### 9.1 `Could not start ffmpeg` or encode fails immediately

- Ensure `ffmpeg` is installed and in `PATH`.
- Confirm required encoders exist (`libx264`, `aac`, etc.).

### 9.2 Codec/encoder not found

- Your FFmpeg build may not include the selected encoder.
- Install a fuller FFmpeg package (or rebuild FFmpeg with needed codecs).

### 9.3 Upload target missing or disabled

- Open **Set Credentials** and click **Save** at least once to create AWS provider entry.
- Confirm bucket is not empty (required).

### 9.4 S3 upload failures

- Verify bucket name and region.
- Verify IAM permissions (`PutObject`, `GetObject`, and if needed `ListBucket`).
- If using `Skip existing`, object read metadata checks still require object read access.

### 9.5 Preview/video playback issues

- Check GStreamer runtime/plugins installation.
- Confirm your platform media stack dependencies are installed.
- If video playback fails to initialize, follow the official GStreamer Rust installation guide:
  - https://github.com/sdroege/gstreamer-rs?tab=readme-ov-file#installation

### 9.6 No audio in output

- If source has no audio stream, app intentionally produces video-only renditions.

## 10. Operational Notes

- Upload worker sends non-playlist files first and playlist files (`.m3u8`) later, reducing partial-playlist issues during upload.
- Overwrite behavior is controlled per upload via UI.
- You can cancel encode/upload from active modal dialogs.
- App keeps recent logs in memory for each operation and trims old lines automatically.
